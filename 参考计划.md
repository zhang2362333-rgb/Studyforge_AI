# StudyForge AI：个人成长版“小黑盒”产品方案

> 文档用途：用于后续项目规划、Vibe Coding、答辩包装、README 扩展、功能拆解与开发排期。  
> 项目仓库：`nanlingyin/Studyforge_AI`  
> 关联思路：StudyForge AI + 小黑盒式社区 + 学习成长档案 + AI 学习助手 + StudyPulse 桌面学习行为感知。

---

## 1. 项目一句话定位

**StudyForge AI 是一个面向大学生与自学者的个人成长社区平台。**

它将知识流、学习路线、AI 摘要、复习卡片、求助悬赏、个人成长档案、社区互动与桌面学习行为感知结合起来，让用户像管理游戏库一样管理自己的技能成长。

更产品化的表达：

> 把学习变成一场有路线、有战绩、有攻略、有队友的长期冒险。

更适合答辩的表达：

> 本项目面向大学生和自学者，构建一个融合 AI 能力的学习成长社区。平台以知识内容为入口，以学习路线和成长档案为核心，通过 AI 摘要、复习卡片、求助讨论、行为感知和成长战报，将用户的阅读、创作、提问、复习和专注行为沉淀为可视化的个人成长数据。

---

## 2. 为什么可以做成“个人提升版小黑盒”

小黑盒类产品的核心不是单纯“发帖”，而是：

1. 有明确的内容对象：游戏。
2. 有个人身份资产：玩家主页、战绩、游戏库。
3. 有社区内容沉淀：攻略、测评、讨论。
4. 有动态与关系链：关注、评论、互动。
5. 有数据反馈：游玩时间、成就、评分、热度。

StudyForge AI 可以对应为：

| 小黑盒 / 游戏社区 | StudyForge AI 对应能力 |
|---|---|
| 游戏库 | 技能库 / 学习路线库 |
| 游戏详情页 | 技能详情页 / 路线详情页 |
| 玩家战绩 | 学习战绩 / 成长档案 |
| 游戏攻略 | 学习攻略 / 技术笔记 / 复盘文章 |
| 游戏测评 | 路线测评 / 课程评价 / 学习资源评价 |
| 正在玩 | 正在学习 / StudyPulse 专注状态 |
| 游戏时长 | 学习时长 / 编码时长 / 阅读时长 |
| 成就系统 | 学习徽章 / 连续学习 / 百赞作者 |
| 社区动态 | 学习动态 / 路线进展 / 复习卡片生成 |
| 开黑组队 | 学习小队 / 自习搭子 / 课设冲刺队 |

因此，StudyForge AI 不应只被描述为“知识内容发布平台”，而应升级为：

> **一个学习者的成长社区、技能资产库和 AI 学习教练系统。**

# 6. 功能一：技能库 / 学习路线库

## 6.1 功能定位

这是“小黑盒化”的关键功能。

小黑盒有“游戏库”，StudyForge 应该有“技能库”。

用户进入平台后，不只是看到一堆文章，而是看到一个个可学习、可收藏、可评价、可追踪进度的技能对象。

### 示例技能

```text

Vue 前端
Spring MVC
MyBatis
MySQL

英语四级
考研数学

AI 产品设计
论文写作
```

## 6.2 技能详情页结构

每个技能详情页可以类似游戏详情页：

| 模块 | 内容 |
|---|---|
| 技能封面 | 大图、标签、难度、热度 |

# 7. 功能二：内容类型系统

## 7.1 当前问题

目前发布页已经很强，但“文章”这个概念太泛。为了更像社区，需要把内容拆成不同类型。

## 7.2 建议内容类型

| 类型 | 英文值 | 用途 |
|---|---|---|
| 攻略 | `GUIDE` | 教程、经验、解决方案 |
| 笔记 | `NOTE` | 学习笔记 |
| 复盘 | `REVIEW` | 学习过程总结 |
| 求助 | `QUESTION` | 遇到问题求解答 |
| 资源 | `RESOURCE` | 工具、链接、书籍、视频 |
| 路线 | `ROADMAP` | 系统学习路径 |
| 动态 | `MOMENT` | 学习状态/进度分享 |

## 7.3 发布页改造

当前“写一篇可以沉淀的学习帖子”可以升级为：

```text
今天想沉淀什么？
[写攻略] [发求助] [记复盘] [做路线] [分享资源] [晒进度]
```

不同类型对应不同模板：

### 攻略模板

```markdown
## 背景

## 适合谁看

## 步骤

1.
2.
3.

## 容易踩坑的地方

## 最后结论
```

### 求助模板

```markdown
## 我想完成什么

## 当前遇到的问题

## 已经尝试过的方法

## 报错信息 / 截图

## 我的环境
```

### 复盘模板

```markdown
## 今天做了什么

## 哪一步最卡

## 我学到了什么

## 下次应该怎么做
```

## 7.4 表结构建议

可以在现有 `posts` 表中增加字段：

```sql
ALTER TABLE posts
    ADD COLUMN content_type VARCHAR(30) NOT NULL DEFAULT 'GUIDE' AFTER category_id;
```

可选值：

```text
GUIDE / NOTE / REVIEW / QUESTION / RESOURCE / ROADMAP / MOMENT
```

# 9. 功能四：StudyPulse 学习行为感知系统

## 9.1 功能定位

StudyPulse 是 StudyForge AI 的“学习状态感知层”。

它可以基于 `fishtime` 的桌面端能力，检测用户当前正在使用的应用，并将其转化为学习行为数据。

一句话定位：

> 自动记录你的学习轨迹，把专注变成可见的成长。

## 9.2 与 fishtime 的关系

`fishtime` 当前是一款桌面时间追踪器，可以自动检测并记录当前正在使用的应用，统计应用使用时长，并生成报告。它使用 Python、PyQt5、SQLite、Matplotlib、psutil、openpyxl、fpdf、pywin32 等技术。

当前 fishtime 具备：

| fishtime 能力 | StudyPulse 改造方向 |
|---|---|
| 实时监控当前应用 | 判断用户正在学习/编码/阅读/摸鱼 |
| 记录应用使用时长 | 生成真实学习时长 |
| SQLite 本地存储 | 本地缓存，联网后同步 |
| 进程白名单 | 隐私白名单 / 不记录名单 |
| 报告导出 | 改为同步到 StudyForge Web |
| 托盘运行 | 桌面学习助手常驻 |
| 多语言/主题 | 保留，增强产品完成度 |

## 9.3 系统架构

不要让 Web 端直接检测用户在做什么。浏览器无法可靠读取系统活动窗口，也不适合做这种能力。

推荐架构：

```text
StudyPulse Desktop Agent
    -> 采集本机应用活动
    -> 本地分类与聚合
    -> 本地 SQLite 缓存
    -> 定时同步到 StudyForge Server

StudyForge Server
    -> 接收活动数据
    -> 归档、分类、统计
    -> 生成每日/每周学习报告
    -> 提供 API 给前端展示

StudyForge Web
    -> 展示今日专注、本周趋势、成长档案、AI 战报、社区动态
```

## 9.4 采集策略

### 不建议上传每秒原始数据

不要上传：

```json
{
  "app": "chrome.exe",
  "windowTitle": "具体网页标题",
  "timestamp": "2026-05-25T20:00:01"
}
```

原因：

1. 隐私风险高。
2. 数据量大。
3. 对产品价值没有明显提升。
4. 容易被用户误解为“监控”。

### 推荐上传聚合数据

每 30 秒或 60 秒聚合一次：

```json
{
  "clientId": "desktop_xxx",
  "startedAt": "2026-05-25T20:00:00",
  "endedAt": "2026-05-25T20:10:00",
  "appName": "Code.exe",
  "appCategory": "CODING",
  "skillCode": "frontend",
  "durationSeconds": 600,
  "confidence": 0.92
}
```

更隐私的默认模式：

```json
{
  "category": "CODING",
  "skillCode": "java_backend",
  "durationSeconds": 600,
  "source": "DESKTOP_AGENT"
}
```

# 15. 隐私与安全设计

StudyPulse 是最有特色的功能，也是最容易被误解的功能，因此必须提前设计隐私边界。

## 15.1 基本原则

```text
用户主动开启。
默认私密。
默认只上传分类后的学习时长。
不上传窗口标题。
不上传敏感内容。
可暂停。
可删除。
可设置白名单和黑名单。
公开动态必须手动确认。
```

## 15.2 产品文案

推荐文案：

> StudyPulse 不会替你审判时间，只是帮你把认真过的痕迹留下来。

或者：

> 只记录你愿意记录的学习轨迹，所有公开都由你决定。

## 15.3 隐私模式

| 模式 | 上传内容 |
|---|---|
| 严格隐私 | 只上传分类时长 |
| 标准模式 | 上传应用名 + 分类时长 |
| 详细模式 | 上传应用名、技能分类、时间段 |
| 本地模式 | 不上传，只在本地显示 |

## 项目亮点

1. **AI 驱动的知识学习闭环**  
   从阅读文章到生成摘要、复习卡片和学习建议，形成完整学习流程。

   把知识内容组织成类似游戏库的技能体系，用户可以选择路线并追踪进度。

4. **StudyPulse 学习行为感知**  
   基于桌面端应用使用记录，自动识别用户学习行为，生成专注数据和 AI 战报。

5. **社区互助与知识沉淀**  
   通过攻略、求助、悬赏、回答采纳和 AI 总结，让问题最终沉淀为可复用知识。

# 25. 最终总结

StudyForge AI 最适合的发展方向不是“普通知识内容平台”，而是：

> **个人成长版小黑盒。**

它的核心不是简单发帖，而是：

```text
知识流负责发现内容
发布器负责沉淀攻略
AI 助手负责提炼知识
复习卡片负责帮助记忆
技能路线负责规划成长
StudyPulse 负责记录真实学习行为
成长主页负责展示战绩
成就系统负责强化反馈
社区动态负责连接用户
求助系统负责解决卡点
```

最终用户体验应该是：

```text
我今天读了什么、写了什么、问了什么、学了多久、在哪个技能上进步了，
StudyForge 都能帮我记录、整理、反馈，并把它变成我的成长档案。
```

这就是 StudyForge AI 区别于普通课程设计项目的核心竞争力。
